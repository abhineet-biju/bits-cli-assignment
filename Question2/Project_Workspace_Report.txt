Question 2 - Secure Project Workspace Setup

Directory Structure Created:
project_workspace/
  plan.txt
  notes.txt

Files Created:
plan.txt contains the project setup plan. notes.txt contains notes explaining the workspace permissions and how they protect project data.

Permissions Before Modification:
drwxr-xr-x 2 root root 4096 Jun 22 21:32 project_workspace

Permissions After Modification:
drwxr-x--- 2 root root 4096 Jun 22 21:32 project_workspace

Ownership Details:
The directory and files were owned by root. The id command showed uid=0(root) gid=0(root) groups=0(root).

Umask Value Used:
0022

Explanation:
The workspace was first created with default directory permissions. Then chmod 750 was applied so the owner had full access, the group had read and execute access, and other users had no access. This protects project data because unauthorized users cannot enter or read the project directory.
